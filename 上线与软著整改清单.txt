====================
P0 阻塞项
====================

[x] 1. 修复前端生产构建失败
目标：
- `app` 目录下 `npm run build` 必须通过
当前已发现问题：
- `app/src/App.vue` 有未使用导入
- `app/src/views/FriendDetailView.vue` 引用了未导入的 `getFriendBackPath`
- `app/src/views/HomeOccasionDetailView.vue` 有未使用类型 `Friend`
- `app/src/services/aiService.ts` 中 `createClient(): never` 与后续 `client.chat...` 类型冲突
验收标准：
- 构建成功
- 无 TypeScript 编译错误

[ ] 2. 立即清理并轮换 AI Key
目标：
- 仓库、数据文件、文档中不再出现真实密钥
当前已发现问题：
- `server/data/system.json`
- `server/data/system.json.migrated`
动作：
- 删除仓库内明文 key
- 更换为占位值或环境变量读取
- 到供应商后台轮换已经泄露的 key
验收标准：
- `rg -n "apiKey|API_KEY"` 不再扫出真实敏感值
当前进度：
- 代码仓库内明文 key 已清理
- 云平台侧旧 key 轮换仍需人工完成

[x] 3. 移除默认弱口令和弱签名密钥
目标：
- 禁止使用默认 `admin` / `change-me` / `change-me-too`
当前位置：
- `server/src/config.mjs`
动作：
- 强制从环境变量读取
- 为空时服务启动失败，不允许静默回退到弱默认值
验收标准：
- 未配置时后端拒绝启动

[ ] 4. 明确正式环境后端地址与 HTTPS 方案
目标：
- App 不能依赖 `http://localhost:9090`
当前位置：
- `app/src/types/settings.ts`
- `app/src/views/MeView.vue`
- `app/src/services/aiService.ts`
动作：
- 区分开发/测试/生产环境
- 配置正式 API 域名
- 启用 HTTPS
验收标准：
- 真机可访问正式后端
- 默认配置不是 localhost
当前进度：
- 前端默认后端地址已改为 `VITE_PROXY_SERVER_URL`
- 代码里不再把 `localhost:9090` 当成默认生产地址
- 正式域名、HTTPS 证书与服务器部署仍待完成

[x] 5. 收紧服务端接口安全边界
目标：
- 不能继续使用原型级开放接口策略
当前问题：
- 全量 `Access-Control-Allow-Origin: *`
- 请求体没有大小限制
- 缺少限流
- 缺少操作审计
- 推荐接口可匿名调用
主要位置：
- `server/src/http.mjs`
- `server/src/index.mjs`
验收标准：
- CORS 白名单可配置
- 请求大小受限
- 关键接口有限流和审计
- 推荐/备份/管理接口鉴权清晰
当前进度：
- 已加入可配置 CORS 白名单
- 已加入请求体大小限制
- 已加入基础内存限流
- `/api/admin/*` 已纳入管理员鉴权前缀
- 已加入关键接口基础审计日志
- 已验证超大请求体返回 `413`

[x] 6. 检查并修复数据越权风险
目标：
- 用户只能访问自己的朋友档案和纪念日
风险点：
- repository 层虽然支持 `listByUserId`
- 但接口层的 `/api/friends`、`/api/memorial-days` 仍是按全局访问/管理态设计
主要位置：
- `server/src/index.mjs`
- `server/src/repositories/friend-repository.mjs`
- `server/src/repositories/memorial-repository.mjs`
验收标准：
- 普通用户接口全部按 userId 隔离
- 管理员接口与普通用户接口彻底分离
当前进度：
- `/api/friends*` 已要求用户登录并按 `userId` 过滤
- `/api/memorial-days*` 已要求用户登录并按 `userId` 过滤
- 已验证用户 A 创建的好友，用户 B 访问返回 `404`

[x] 7. 关闭或评估 Android 自动备份风险
目标：
- 朋友档案与画像数据不能默认随系统备份外流
当前位置：
- `app/android/app/src/main/AndroidManifest.xml`
当前问题：
- `android:allowBackup="true"`
验收标准：
- 生产包策略明确
当前进度：
- 已关闭 `android:allowBackup`
- 已增加 `backup_rules.xml` 与 `data_extraction_rules.xml`
- 当前策略为默认不参与云备份和设备迁移备份
